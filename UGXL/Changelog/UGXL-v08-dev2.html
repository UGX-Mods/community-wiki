Summary

I'm happy to announce that the next developer version (#2)  is available.
Most of the reported issues were fixed and a couple of new improvements and additions made it into the release as well.

With #2, the updater will be tested (which hopefully works)

Please follow the Manual Patching Guide below before you launch UGXL!

 

Important highlights from this release

  1. Awesomium (WebBrowser, MapWebView) does (finally) receive key input.
  2. Files can be downloaded not with the need of an extension in the URL. (was needed for profile pictures)
  3. WaW is handled properly now (and no annoying "Run in safemode" popup anymore)
  4. Problems with account data has been resolved
  5. Rightclick context menu for the map items has been added.
    Also (unmentioned change) its connected to the map state, so it will change if map state changes as well.
  6. Group function does work now (rename, delete, etc)
  7. UGXL works on Vista now (yeah it's apparently still a thing)
  8. Added new file watcher system and auto-reload of current theme added.
  9. Treelist item do resize their label depending on the size now. Also the missing context menu was added.
    (The button next to the search acts currently as the "group context menu button" because a rightclick on an empty space is hard
  10. Fixed high CPU usage (from ~30 % to ~14)

Notes

Another unmentioned change is the pre-revision check.
Currently it will remove any existing CrashRpt (because older ones will break crashrpt! - and it will avoid uploading of possible resolved issues already)

Also the <root>/bin/7z.exe will be removed (deprecated through 7za.exe in the manual update and new versions)

The web backend has been improved. Page load should be noticeable faster now. (also the map reviews do slide now if there are >3 - it was disabled, and I don't know why)
CrashRPT(.php) and Report(.php) has been improved a lot. (updated 1y old code, more information written to ticket)
Auth(.php) has been improved as well.

Testing

This release introduced a couple of new features / additions, which need to be well tested to ensure it works with the rest of the application.

Especially the improved context menu should be tested. ( Note:  on map state change, the context menu should either update or disappear)
