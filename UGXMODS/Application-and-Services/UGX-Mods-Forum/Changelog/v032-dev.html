Notes

This is a developer version and sharing with someone else is obviously disallowed.

Only users with MySQL access can actually use the SQL queries.

How to use the SQL Developer Mode

You can use this parameter 

-sqldev

which will go directly into that mode (to create SQL quieres without a mod / map)

 

Otherwise, select the mod / map folder in step one, validate and create the archive.

Once this is done, fill in the missing information for the first table and hit the 1. button. This will copy an SQL query to your clipboard.

 

Execute the SQL query from your clipboard within the SMF Database (this is important, be in that database in phpmyadmin and click on SQL). Paste the query and press Go.

After inserting, copy the mid (Map ID) from the created row and add it in the proper input field in UGXMAPP. (the get rid of the placeholder text, either select all or use the up / down key or the buttons, then you can paste the number)

Usually for a new mod, the revision is 1 - but for multiple versions the revision needs to be incremented. For example version 1.0 would be rev1, while 1.1 would be rev2!

The version input is the text form, visible to the user instead of the revision (which is used internally!)

 

If this version is a beta version, check it and add a strong random generated password. The UGX Launcher and later the UGX MapValidator can decrypt the archive with this password then.

 

Description and Changelog can be HTML. Description will describe the general map, while the Changelog lists all the changes of the version.

Please keep these information, including the release date as close as possible and take it seriously! This will ensure good quality.

To convert the text to html, used this: http://www.textfixer.com/html/convert-text-html.php (and use <br />: Use line break tags only)

 

The status public will show the mod / map in UGXL / UGX Play, while private will keep it hidden for everyone for further processing.
If it's public, make sure that the archive is uploaded already or users will experience issues with downloading.

 

If you created the compressed archive, the generated hash will be inserted into the correct cell as well!

 

The last query is the creators table, which will list creator, submitter and co.
The coloums are: "uid" (User ID of SMF user), "role" (role of the UGC) - select the proper one from the combobox, "Description" (what this user did)
The data will be then used to list creators and put all ppl to the credits list.

 

If an user doesn't exist as UGX Account, use the UID 0 - this will be a special ID and write the username simply in the description as well for example "Thanks to ABZ for XYZ" or "Dolphin for helping me with eee ee eeeeee"

 

Once all queries are inserted, you are done

The 7z needs to be uploaded to:

/home/ugx-mods.com/public_html/ugxl/maps/<modname>/

a preview.jpg will be used for preview. (size is currently: 294x172) 

/home/ugx-mods.com/public_html/ugxl/maps/<modname>/media

can contain media files for the slider.
The name needs to be 1, 2, 3 and so one (for the oder)
t_1 would be the thumbnail (good for youtube videos)

 

review other added maps for reference (requiem / comosea)
